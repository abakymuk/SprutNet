# T16. Человечные ошибки и сообщения

**Goal:** Понятные пользователю ошибки вместо кодов и стеков.

**Scope:**
- In: словарь сообщений для 400/404/429/5xx, кнопка-фоллбек на моки, повтор запроса.
- Out: доверие и предсказуемость поведения.

**AC (Gherkin):**
- Given 400/404, Then показываю "Не найдено расписаний по выбранным портам/датам…".
- Given 429, Then "Слишком много запросов. Повторим автоматически.". 
- Given 5xx, Then "Проблема на стороне поставщика…" + кнопка моки.

**DoD:**
- [x] Сообщения локализованы/скрыты за словарём.
- [x] Единый компонент ErrorState/Skeleton.
- [x] e2e: сценарии ошибок покрыты.

### 📊 **Примеры человечных сообщений:**

**400 Bad Request:**
```
Заголовок: "Неверный запрос"
Сообщение: "Проверьте правильность введенных данных и попробуйте снова."
Подсказка: "Убедитесь, что все поля заполнены корректно."
Действие: "Исправить данные"
```

**404 Not Found:**
```
Заголовок: "Данные не найдены"
Сообщение: "По вашему запросу ничего не найдено."
Подсказка: "Попробуйте изменить параметры поиска или даты."
Действие: "Изменить параметры"
```

**429 Rate Limited:**
```
Заголовок: "Слишком много запросов"
Сообщение: "Превышен лимит запросов к серверу."
Подсказка: "Мы автоматически повторим запрос через несколько секунд."
Действие: "Повторить сейчас"
```

**500 Server Error:**
```
Заголовок: "Ошибка сервера"
Сообщение: "Произошла внутренняя ошибка на сервере."
Подсказка: "Попробуйте позже или используйте демо-данные."
Действие: "Использовать демо-данные"
```
