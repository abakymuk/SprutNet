# T13. Сохранить моки как фоллбек + фича-флаги

**Goal:** Возможность мгновенно переключиться на моки при инцидентах/лимитах API.

**Scope:**
- In: проверка `FEATURE_MAERSK`, возврат `{ items: MOCK_* }` при выключенном флаге, fallback-кнопка в UI ("Показать демо-данные").
- Out: устойчивость демо и стендапов.

**AC (Gherkin):**
- Given FEATURE_MAERSK=false, When вызываю `/api/schedules`, Then получаю мок-данные.
- Given 5xx/429, When пользователь нажимает "Показать моки", Then UI рендерит мок-результат.

**DoD:**
- [x] Флаги учтены в каждом route.
- [x] Кнопка-фоллбек в UI при фатальной ошибке.
- [x] Документация: как переключать режимы.
