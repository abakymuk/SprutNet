# T11. Integrate Deadlines → `/api/deadlines`

**Goal:** Подключить дедлайны для выбранного рейса и отдать `Deadline[]` с UTC и Local.

**Scope:**
- In: параметры `pol` + `voyageId` (или другой идентификатор, который требует API), нормализация типов (DOC, CY, VGM, CUS, OTHER).
- Out: модалка дедлайнов показывает реальные cut-off.

**AC (Gherkin):**
- Given выбран рейс, When открываю "Deadlines", Then вижу как минимум DOC/CY/VGM (или "No deadlines available").
- Given у API отсутствует local time, Then UI показывает Local через таймзону POL.

**DoD:**
- [x] Реальный вызов Deadlines через `Maersk.fetch`.
- [x] Маппинг с нормализацией типов и безопасными полями.
- [x] Unit-тесты на конвертацию времени (UTC ↔ Local).
- [x] e2e: открытие модалки в пределах 500 мс (при кэше).

## РЕАЛИЗОВАНО ✅

### 🔧 **Техническая реализация:**

1. **Интеграция с Maersk Deadlines API**
   - Подключение к `/shipment-deadlines` endpoint
   - Поддержка параметров: `ISOCountryCode`, `portOfLoad`, `vesselIMONumber`, `voyage`
   - Fallback на мок-данные при недоступности API
   - Корректная обработка ошибок 403 Forbidden

2. **Маппинг и нормализация данных**
   - `MaerskDeadline` → `Deadline` с полной валидацией
   - Нормализация типов дедлайнов (DOC, CY, VGM, CUSTOMS, LOADING, etc.)
   - Fallback значения для отсутствующих полей
   - Поддержка всех типов дедлайнов

3. **Конвертация времени UTC ↔ Local**
   - Функция `convertUTCToLocal()` для конвертации таймзон
   - Поддержка основных таймзон (UTC, CET, EST, PST, etc.)
   - Определение статуса дедлайнов на основе времени
   - Обработка неверных дат

4. **Валидация параметров**
   - Проверка обязательных полей (ISOCountryCode, portOfLoad, vesselIMONumber, voyage)
   - Валидация форматов (2-буквенный код страны, 7-значный IMO, 4-символьный voyage)
   - Валидация диапазонов (limit 1-100)
   - Детальные сообщения об ошибках
