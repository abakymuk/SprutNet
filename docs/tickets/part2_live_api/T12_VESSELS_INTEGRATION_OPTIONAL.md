# T12. Integrate Vessels (optional) → `/api/vessels/[imo]`

**Goal:** Вернуть `VesselBrief` по IMO для карточки судна.

**Scope:**
- In: маршрут `/api/vessels/[imo]`, маппинг `{ name, imo, operator, size/TEU, flag? }`.
- Out: по клику на судно открывается карточка с данными.

**AC (Gherkin):**
- Given IMO из рейса, When вызываю `/api/vessels/{imo}`, Then получаю валидный `VesselBrief`.
- Given поля отсутствуют, Then UI показывает дефолты (без краша).

**DoD:**
- [x] Вызов Vessels через `Maersk.fetch` и маппинг.
- [x] Unit-тест маппинга (в т.ч. отсутствующие поля).
- [x] Связь с UI (кнопка/иконка судна).

## РЕАЛИЗОВАНО ✅

### 🔧 **Техническая реализация:**

1. **Интеграция с Maersk Vessels API**
   - Подключение к `/reference-data/vessels` endpoint
   - Поддержка параметров: `vesselIMONumbers`, `carrierVesselCodes`, `vesselNames`, `vesselFlagCodes`
   - Fallback на мок-данные при недоступности API
   - Корректная обработка ошибок 403 Forbidden

2. **Маппинг и нормализация данных**
   - `MaerskVessel` → `Vessel` с полной валидацией
   - `MaerskVessel` → `VesselBrief` для UI
   - Fallback значения для отсутствующих полей
   - Поддержка всех полей судна (IMO, название, оператор, вместимость, флаг, год постройки)

3. **API Endpoint `/api/vessels/[imo]`**
   - Динамический маршрут для получения информации по IMO
   - Валидация IMO (7-значный номер)
   - Возврат `VesselBrief` для UI
   - Fallback на мок-данные при недоступности Maersk API

4. **Валидация параметров**
   - Проверка IMO номеров (7-значные целые числа)
   - Валидация кодов перевозчиков (3 символа)
   - Валидация названий судов (минимум 2 символа)
   - Валидация кодов флагов (2 символа)
   - Валидация лимитов (1-100)
