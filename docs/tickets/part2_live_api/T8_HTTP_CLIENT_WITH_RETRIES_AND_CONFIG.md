# T8. HTTP-клиент с гибким хедером и ретраями

**Goal:** Ввести единый клиент `Maersk.fetch()` с поддержкой кастомного AUTH-хедера, кэша и бэкоффа.

**Scope:**
- In: файл `apps/web/lib/maersk.ts`, конфиг AUTH-хедера, ретраи (429/5xx), базовый in-memory LRU-кэш (TTL из ENV).
- Out: обращения к API из UI.

**AC (Gherkin):**
- Given неверный ключ, When вызываю `Maersk.fetch`, Then получаю контролируемую ошибку и дружелюбное сообщение в UI.
- Given 429 от API, When повторный вызов, Then клиент делает до 3 попыток с экспоненциальной задержкой.
- Given одинаковый запрос в течение TTL, Then ответ берётся из кэша.

**DoD:**
- [ ] Реализован единый клиент с AUTH-хедером из ENV.
- [ ] Ретраи для 429/5xx, логируется `api_retry`.
- [ ] Простой LRU/Map-кэш с TTL, логируется `cache_hit`.
- [ ] Покрыт unit-тестами на ретраи/кэш (минимум 3 кейса).
