# UC2: Просмотр деталей рейса

## JTBD (Job To Be Done)
**«Мне нужно выбрать оптимальный рейс и обосновать свой выбор клиенту/руководству»**

## Описание
UC2 позволяет пользователям просматривать детальную информацию о рейсах, сравнивать их между собой и получать обоснование для выбора оптимального варианта.

## Основные функции

### 1. Открытие карточки рейса
- **Триггер**: Нажатие кнопки "Подробнее" или "Детали рейса" в списке рейсов
- **Результат**: Открытие модального окна с детальной информацией

### 2. Просмотр детальной информации
Пользователь видит:
- **Судно**: название, IMO номер, вместимость, год постройки, флаг
- **Оператор**: название перевозчика, код, рейтинг надежности, контакты
- **ETD/ETA**: даты отправления и прибытия с учетом часовых поясов
- **Transit days**: время в пути в днях
- **Цены**: тарифы по типам контейнеров, включая доплаты

### 3. Сравнение рейсов
- Позиция рейса среди всех доступных вариантов
- Рейтинг по цене, скорости, надежности
- Альтернативные рейсы с их характеристиками
- Общий балл рейса (0-100)

### 4. Обоснование выбора
- Ключевые преимущества выбранного рейса
- Рекомендации для клиента
- Готовое обоснование для руководства

## Компоненты

### RouteDetailsModal
Основной компонент для отображения детальной информации о рейсе.

**Файл**: `apps/web/src/components/planner/RouteDetailsModal.tsx`

**Пропсы**:

**Табы**:
1. **Обзор** - основная информация и рекомендации
2. **Расписание** - детальное расписание и важные даты
3. **Судно** - информация о судне и операторе
4. **Сравнение** - позиция среди других рейсов
5. **Обоснование** - ключевые преимущества и готовое обоснование

## API интеграция

### Получение данных рейса
```typescript
// API endpoint для получения рейсов
GET /api/routes/search?originPort=SHANGHAI&destinationPort=USLAX&...

// Структура ответа
{
  routes: RouteOption[],
  recommendations: RouteRecommendation[],
  source: "maersk" | "cache" | "fallback"
}
```

### Структуры данных

#### RouteOption
```typescript
interface RouteOption {
  id: string;
  carrierName: string;
  carrierCode: string;
  voyageNumber: string;
  originPort: PortRef;
  destinationPort: PortRef;
  departureDate: string;
  arrivalDate: string;
  transitTime: number;
  vessel: {
    name: string;
    imoNumber: string;
    capacity: number;
    builtYear?: number;
    flag?: string;
  };
  rates: Rate[];
  score?: number;
}
```

#### RouteRecommendation
```typescript
interface RouteRecommendation {
  type: "earliest" | "shortest" | "cheapest" | "balanced" | "reliable";
  title: string;
  description: string;
  icon: React.ElementType;
  color: string;
  score: number;
}
```

## Алгоритмы

### Расчет общего балла
```typescript
const overallScore = Math.round(
  (100 - (priceRank - 1) * 10) * 0.3 +      // Цена (30%)
  (100 - (speedRank - 1) * 10) * 0.3 +      // Скорость (30%)
  (100 - (reliabilityRank - 1) * 10) * 0.2 + // Надежность (20%)
  (100 - (popularityRank - 1) * 10) * 0.2    // Популярность (20%)
);
```

### Генерация рекомендаций
- **Самый ранний**: если рейс отправляется раньше всех
- **Самый быстрый**: если самый короткий транзит
- **Самый дешевый**: если лучшая цена
- **Сбалансированный**: если общий балл > 80
- **Надежный**: если позиция по надежности ≤ 3

## Пользовательский опыт

### Workflow
1. Пользователь находит рейсы через UC1
2. Нажимает "Подробнее" на интересующем рейсе
3. Просматривает детальную информацию по табам
4. Сравнивает с другими рейсами
5. Получает готовое обоснование выбора
6. Принимает решение и выбирает рейс

### Ключевые преимущества
- **Полная информация**: все необходимые данные в одном месте
- **Сравнение**: понимание позиции рейса среди альтернатив
- **Обоснование**: готовые аргументы для клиента/руководства
- **Удобство**: структурированная информация по табам

## Производительность

### Оптимизации
- Ленивая загрузка данных судна через `VesselCard`
- Кэширование результатов сравнения
- Мемоизация вычислений метрик

### Метрики
- Время открытия модального окна: < 100ms
- Время загрузки данных судна: < 500ms
- Время расчета метрик: < 50ms

## Ошибки и обработка

### Типичные ошибки
- Отсутствие данных о судне
- Недоступность API перевозчика
- Отсутствие цен

### Fallback стратегии
- Использование базовой информации о судне
- Показ контактов из справочника
- Расчет примерных цен на основе исторических данных
