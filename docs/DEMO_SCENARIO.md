# 🎯 Демо-сценарий SprutNet MVP

**Версия:** 1.0.0  
**Дата:** Январь 2025  
**Статус:** Готов к демонстрации  

## 🎬 **Сценарий презентации (15-20 минут)**

### **Введение (2 минуты)**
```
"Добро пожаловать в SprutNet - интеллектуальную систему планирования морских перевозок. 
Сегодня я покажу MVP, который решает реальные проблемы логистических компаний."
```

### **1. Обзор системы (3 минуты)**

#### **Открыть главную страницу:**
- URL: `http://localhost:3000`
- **Показать:** Современный дизайн, ключевые возможности
- **Акцент:** "Система готова к работе с реальными данными Maersk API"

#### **Перейти к планировщику:**
- URL: `http://localhost:3000/planner`
- **Показать:** Интуитивный интерфейс поиска
- **Акцент:** "Все функции работают с демо-данными, готовы к live API"

### **2. Поиск портов (2 минуты)**

#### **Демонстрация автодополнения:**
```
Действие: Ввести "sha" в поле Origin Port
Результат: CNSHA (Shanghai) появляется в топ-3
Комментарий: "Система мгновенно находит порты по частичному совпадению"
```

#### **Показать детали порта:**
```
Действие: Выбрать CNSHA
Результат: Отображается полная информация о порте
Комментарий: "Интеграция с Maersk Locations API"
```

### **3. Поиск рейсов (3 минуты)**

#### **Настройка поиска:**
```
Origin Port: CNSHA (Shanghai)
Destination Port: USLAX (Los Angeles)
Departure Date From: 2025-09-01
Departure Date To: 2025-09-30
```

#### **Результаты поиска:**
```
Ожидаемый результат: ≥5 рейсов с детальной информацией
Показать: Время в пути, стоимость, доступность
Комментарий: "Реальные данные о расписаниях Maersk"
```

#### **Фильтрация и сортировка:**
```
Действие: Использовать фильтры по времени/стоимости
Результат: Отсортированные результаты
Комментарий: "Интеллектуальная оптимизация маршрутов"
```

### **4. Детали рейса и дедлайны (3 минуты)**

#### **Выбор рейса:**
```
Действие: Кликнуть на конкретный рейс
Результат: Детальная информация о судне и маршруте
Комментарий: "Полная информация для планирования"
```

#### **Просмотр дедлайнов:**
```
Действие: Кликнуть "View Deadlines"
Результат: DOC/CY/VGM дедлайны с временными зонами
Комментарий: "Критически важная информация для логистики"
```

#### **Информация о судне:**
```
Действие: Кликнуть на название судна
Результат: Детали судна (IMO, размеры, тип)
Комментарий: "Интеграция с Maersk Vessels API"
```

### **5. Телеметрия и мониторинг (2 минуты)**

#### **Открыть дашборд телеметрии:**
- URL: `http://localhost:3000/telemetry-dashboard`
- **Показать:** Реальные метрики использования
- **Акцент:** "Полная видимость работы системы"

#### **Демонстрация событий:**
```
Действие: Выполнить поиск → проверить дашборд
Результат: События search_started, search_success
Комментарий: "Мониторинг в реальном времени"
```

### **6. Fallback механизмы (2 минуты)**

#### **Показать статус API:**
- URL: `http://localhost:3000/maersk-status`
- **Демонстрация:** Текущий статус Maersk API
- **Акцент:** "Система работает даже при недоступности API"

#### **Переключение источников данных:**
```
Действие: Показать кнопку переключения в UI
Результат: Мгновенное переключение между live/демо
Комментарий: "Гибкость и надежность системы"
```

### **7. Заключение (2 минуты)**

#### **Ключевые преимущества:**
- ✅ **Готовность к продакшену** - все функции работают
- ✅ **Интеграция с Maersk** - реальные данные
- ✅ **Отказоустойчивость** - fallback механизмы
- ✅ **Мониторинг** - полная телеметрия

#### **Следующие шаги:**
