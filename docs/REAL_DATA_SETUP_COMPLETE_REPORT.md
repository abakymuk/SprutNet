## 📊 Режимы работы

### 1. Смешанный режим (по умолчанию)

```env
FEATURE_MAERSK=true
FEATURE_REAL_DATA_ONLY=false
```

**Поведение:**
- ✅ Использует реальные данные от Maersk API
- ✅ При ошибках API возвращает fallback данные
- ✅ Подходит для разработки и тестирования

### 2. Режим только реальных данных

```env
FEATURE_MAERSK=true
FEATURE_REAL_DATA_ONLY=true
```

**Поведение:**
- ✅ Использует исключительно реальные данные
- ❌ При ошибках API возвращает HTTP 503 ошибку
- ✅ Подходит для продакшена

## 🔍 Анализ текущего состояния

### ✅ Работающие компоненты
1. **API Health Check** - работает корректно
2. **API Ports Search** - получает реальные данные от Maersk API
3. **Кэширование** - система кэша настроена и работает
4. **Логирование** - улучшено логирование ошибок

### ⚠️ Проблемные компоненты
1. **API Vessels** - возвращает "Доступ запрещен"
2. **API Schedules** - возвращает пустые данные
3. **Maersk API** - частично недоступен

### 🔧 Возможные причины проблем
1. **Ограничения API ключей** - возможно, ключи имеют ограничения на определенные endpoints
2. **Rate limiting** - превышение лимитов запросов
3. **Изменения в API** - Maersk мог изменить структуру API
4. **Аутентификация** - проблемы с токенами доступа

## 🚀 Готово к использованию

```bash

./scripts/test-real-data.sh
```

### Переключение режимов
```bash
# Включение режима только реальных данных
echo "FEATURE_REAL_DATA_ONLY=true" >> .env.local

# Отключение режима только реальных данных
# Удалите или закомментируйте FEATURE_REAL_DATA_ONLY=true в .env.local
```
